# Copywriting

## Core Principles
1. **Clarity over cleverness** — If you must choose, choose clear
2. **Benefits over features** — "Save 4 hours/week" beats "automated reporting"
3. **Specificity over vagueness** — "500+ teams" beats "many companies"
4. **Customer language** — Mirror the words they use in reviews and support tickets
5. **One idea per section** — Don't overload

## Headline Formulas
```
[Outcome] without [pain point]
→ "Ship features faster without breaking production"

Stop [bad thing]. Start [good thing].
→ "Stop guessing. Start knowing."

The [category] that [unexpected benefit]
→ "The CRM that actually saves you time"

[Number] [people/companies] [action] with [product]
→ "2,000+ founders track metrics with MetricFlow"

[Do thing] in [surprisingly short time]
→ "Launch your SaaS in one weekend"
```

## CTA Button Copy
Bad: "Submit", "Click Here", "Learn More"
Good:
- "Start free trial"
- "Get started — it's free"
- "See it in action"
- "Create your first [thing]"
- "Try for free — no credit card"

## Feature Description Pattern
```
**[Benefit-focused title]**
[One sentence explaining what it does and why it matters.]
```

Example:
```
**Ship 3x faster with type-safe APIs**
tRPC connects your frontend to backend with zero manual type definitions.
Change a field on the server — your React components know instantly.
```

## Email Subject Lines
- Keep under 50 characters
- Create curiosity or urgency
- Personalize when possible
- Test: would YOU open this?

Good: "Your weekly report is ready"
Good: "3 features you missed this week"
Bad: "Newsletter #47 — February Update"

## Error Messages (UX copy)
Bad: "Error 422: Validation failed"
Good: "That email is already registered. Want to log in instead?"

Pattern: **What happened** + **What to do next**
